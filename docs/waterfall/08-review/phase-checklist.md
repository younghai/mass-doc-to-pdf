## G0 — 기획 게이트

**목적:** 서비스 방향성과 성공 지표를 조직이 합의했는지 확인한다.

| 번호 | 항목 | 기준 | 상태 |
| --- | --- | --- | --- |
| G0-1 | 서비스 기획서 승인 | `00-planning/service-planning.md` 문서 작성 및 검토 완료 | ✅ |
| G0-2 | 비즈니스 정책서 승인 | `00-planning/business-policy.md` 승인 | ✅ |
| G0-3 | KPI 정의 완료 | 전환 성공률, 업타임, 응답 시간 목표 수치 확정 | ✅ |
| G0-4 | 지원 포맷 정의 | HWP/HWPX/DOC/DOCX/PPT/PPTX/XLS/XLSX 범위 확정 | ✅ |
| G0-5 | 배포 방식 결정 | Docker Compose / Standalone 2종 방식 결정 | ✅ |

**G0 통과 기준:** 위 5개 항목 모두 체크.

## G1 — 요구사항 게이트

**목적:** 시스템이 구현해야 할 기능/비기능 요구사항이 완전하고 추적 가능한지 확인한다.

| 번호 | 항목 | 기준 | 상태 |
| --- | --- | --- | --- |
| G1-1 | SRS 베이스라인 | `01-requirements/` 문서 작성 완료 | ✅ |
| G1-2 | RTM (요구사항 추적 매트릭스) 완성 | 각 요구사항이 설계/구현 항목과 매핑됨 | ✅ |
| G1-3 | 유스케이스(UC) 승인 | 업로드, 변환, 다운로드, 인증 UC 작성 완료 | ✅ |
| G1-4 | 비기능 요구사항 정의 | 응답 시간, 가용성, 보안, 파일 크기 한도 | ✅ |
| G1-5 | 이해관계자 승인 | 기획/개발 동의 | ✅ |

**G1 통과 기준:** 위 5개 항목 모두 체크. SRS 미완성 시 G1 통과 불가.

## G2 — 시스템 설계 게이트

**목적:** 전체 시스템 구조가 요구사항을 충족하고 구현 가능한지 확인한다.

| 번호 | 항목 | 기준 | 상태 |
| --- | --- | --- | --- |
| G2-1 | 아키텍처 설계 승인 | `02-system-design/` 문서 작성, 서비스 구성도 포함 | ✅ |
| G2-2 | DB 설계 승인 | Prisma 스키마 기반 ERD 검토 완료 | ✅ |
| G2-3 | API 설계 승인 | REST API 엔드포인트 목록, 요청/응답 스키마 정의 | ✅ |
| G2-4 | 인증 설계 승인 | Google OAuth + 세션/JWT 방식 결정 | ✅ |
| G2-5 | 스토리지 설계 승인 | S3 호환 MinIO / 로컬 파일 이중 방식 설계 | ✅ |
| G2-6 | 배포 아키텍처 승인 | Docker Compose / Standalone 아키텍처 문서화 | ✅ |

**G2 통과 기준:** 위 6개 항목 모두 체크. 아키텍처 미승인 시 G3 진입 금지.

## G3 — 상세 설계 게이트

**목적:** 구현에 필요한 세부 설계가 완성되었는지 확인한다.

| 번호 | 항목 | 기준 | 상태 |
| --- | --- | --- | --- |
| G3-1 | 엔진 체인 설계 승인 | HWP 체인(rhwp→sidecar→builtin), Office 체인(gotenberg→builtin) | ✅ |
| G3-2 | 상태머신 검증 | Job 상태 (pending→running→done/failed) 전이 다이어그램 검토 | ✅ |
| G3-3 | 품질 게이트 설계 | QualityReport 스키마 v1, 판정 기준 (pass/warn/fail) | ✅ |
| G3-4 | 큐 설계 | USE_QUEUE 분기, claimNext, stuck-running reaper 로직 | ✅ |
| G3-5 | 보안 설계 | CSRF Origin 검증, rate limit, trustProxy, DEV_AUTH 비활성화 | ✅ |
| G3-6 | 에러 처리 설계 | 4xx/5xx 응답 코드 표준화, 에러 메시지 형식 | ✅ |

**G3 통과 기준:** 위 6개 항목 모두 체크.

## G6 — 배포 게이트

**목적:** 운영 환경 배포가 안전하게 완료되었는지 확인한다.

| 번호 | 항목 | 기준 | 상태 |
| --- | --- | --- | --- |
| G6-1 | 스모크 테스트 통과 | 배포 후 업로드→변환→다운로드 성공 | ✅ |
| G6-2 | 헬스체크 통과 | `/health` → `{"status":"ok"}` | ✅ |
| G6-3 | 보안 체크리스트 통과 | `env-security.md` 체크리스트 모두 체크 | ✅ |
| G6-4 | `DEV_AUTH=0` 확인 | 운영 배포에 개발 자동 로그인 비활성화 | ✅ |
| G6-5 | `AUTH_SECRET` 강도 확인 | 32바이트 이상 무작위값 | ✅ |
| G6-6 | `WEB_ORIGIN` 운영 URL | 운영 도메인으로 설정됨 | ✅ |
| G6-7 | HTTPS 적용 확인 | SSL/TLS 종단 구성 완료 | ✅ |
| G6-8 | DB 백업 확인 | 배포 전 백업 완료 | ✅ |
| G6-9 | 롤백 절차 준비 | 이전 버전 보관, 롤백 명령 확인 | ✅ |
| G6-10 | 운영 문서 완비 | 배포 계획서, 운영 가이드, 보안 가이드 완료 | ✅ |

**G6 통과 기준:** 위 10개 항목 모두 체크. G6-4 (DEV_AUTH) 또는 G6-7 (HTTPS) 미통과 시 배포 차단.
